# cosmo-scan

Микросервисная платформа для загрузки, валидации и автоматического анализа файлов домашних заданий
от студентов.

![Схема взаимодействия](/readme-resources/scheme.jpg)

Предоставляет отчеты о проверке и единую точку входа через API Gateway.

## file-storing-service

Микросервис file-storing-service отвечает за хранение доменных сущностей - студентов и домашних
работ.  
Запускается в отдельном контейнере на порту 8080.  
Эндпоинты, через которые можно обратиться к микросервису - "/api/students/..." и "/api/files/...".

Структура микросервиса:

Domain - Repository - Service (отдельно FileStorageService) - Controller

Основной функционал микросервиса - при загрузке файла отправляется "сигнал"-триггер на микросервис
по анализу документов для проведения этой операции.

## file-analysis-service

Микросервис file-analysis-service отвечает за анализ файлов (домашних работ) - проверяет формат
файлов и их размер и предоставляет отчет о проведенной проверке.

Запускается в отдельном контейнере на порту 8082.  
Эндпоинты, через которые можно обратиться к микросервису - "/api/analysis".

Структура микросервиса:

Domain - Repository - Service - Controller

Основной функционал микросервиса - при поступлении сигнала осуществляется проверка файла.  
В результате доступны для просмотра отчеты по проведенному анализу.

## api-gateway

Микросервис api-gateway выполняет роль оркестратора (фасада), принимающего команды, которые могут
отправляться на оба микросервиса:

- file-storing-service
- file-analysis-service  
  и решает на какой из них отправить команду.

Основной функционал расположен в конфигураторе application.yml.

Микросервис запускается в отдельном контейнере на порту 8081.

**OpenAPI/Swagger** доступен после запуска:  
Storing: http://localhost:8080/swagger-ui.html  
Analysis: http://localhost:8082/swagger-ui.html

## Endpoints

### Endpoints file-storing-service

Для студентов

| Метод    |         Путь         | Действие                             |
|----------|:--------------------:|--------------------------------------|
| `POST`   |   `/api/students`    | Создать студента (тело: имя строкой) |
| `GET`    |   `/api/students`    | Список всех студентов                |
| `GET`    | `/api/students/{id}` | Данные одного студента               |
| `DELETE` | `/api/students/{id}` | Удалить студента                     |

Для домашних работ

| Метод    | Путь                       | Действие                                                          |
|----------|----------------------------|-------------------------------------------------------------------|
| `POST`   | `/api/files/upload`        | Загрузить файл + создать запись о ДЗ + запустить анализ           |
| `GET`    | `/api/files`               | Список всех домашних работ                                        |
| `GET`    | `/api/files/{id}`          | Информация о конкретной работе                                    |
| `GET`    | `/api/files/{id}/download` | Скачать файл (возвращает байты + заголовок `Content-Disposition`) |
| `DELETE` | `/api/files/{id}`          | Удалить файл и запись о ДЗ                                        |

### Endpoints file-analysis-service

| Метод  | Путь                                           | Действие                                                |
|--------|------------------------------------------------|---------------------------------------------------------|
| `POST` | `/api/analysis/trigger`                        | Запустить анализ файла (тело: `{homeworkId, fileName}`) |
| `GET`  | `/api/analysis/reports`                        | Список всех отчетов                                     |
| `GET`  | `/api/analysis/reports/{reportId}`             | Детали конкретного отчета                               |
| `GET`  | `/api/analysis/homeworks/{homeworkId}/reports` | Отчеты по конкретной homework                           |
| `POST` | `/api/analysis/word-cloud`                     | Получить файл с облаком слов из случайного .txt файла   |                                                |                                                         |

## Облако слов

Дополнительно в file-analysis-service имеется возможность отправки запроса на
сервис https://quickchart.io/wordcloud для получения файла-изображения с облаком слов - наиболее
встречающихся слов в файле.  
На генерацию облака слов отправляется случайный .txt файл из хранилища, который успешно прошел
проверку в сервисе analysis-service.

![Облако слов](/readme-resources/words-cloud.jpg)

Примеры файлов для загрузки лежат в папке sample-files.

## Отказоустойчивость

CircuitBreaker и fallbackUri в **API Gateway** защищают от каскадных сбоев.

## Запуск и проверка

**Запуск**

```bash
docker-compose up -d --build
```  

**Проверка работоспособности**

```bash
http://localhost:8081/actuator/health
```  

**Посмотреть активные маршруты**

```bash
http://localhost:8081/actuator/gateway/routes
```

## Тесты (юнит, интеграционные)

Покрытие тестами кода составляет более 80 %. Для проверки - генерация отчета о покрытии:

```bash
gradle jacocoTestReport
```

Отчеты будут лежать в **build/reports/jacoco/test/html/index.html** в папках у каждого микросервиса.

## Функциональное тестирование

Для проверки корректной работы сервисов можно проверить работу коллекций при помощи **POSTMAN**.  
Коллекции необходимо импортировать из папки **postman** в корне репозитория.
